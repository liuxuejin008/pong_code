# 产品功能文档：全部迭代列表页

## 1. 文档概述
*   **功能模块**：项目管理 / 迭代管理
*   **页面名称**：全部迭代 (All Iterations)
*   **功能目标**：提供一个中心化的视图，供项目成员和管理者查看、筛选、搜索及管理项目中的所有迭代（Sprint）及其状态。

## 2. 页面布局 (UI Structure)

页面主要由三个部分组成：**顶部全局导航栏**、**二级操作/筛选栏**、**数据列表区域**。

### 2.1 顶部全局导航栏
*   **左侧导航菜单**：
    *   包含：概览、规划、需求、缺陷、工作项、**迭代 (当前选中)**、发布、基线、更多。
    *   *交互*：点击不同Tab跳转至对应模块。
*   **右侧工具栏**：
    *   **全局搜索框**：支持快捷键 `⌘+K`，支持模糊搜索。
    *   **新建按钮 (+)**：快速创建对象。
    *   **帮助 icon (?)**：跳转帮助文档。
    *   **通知中心**：显示未读消息数（红点数字，如图中“13”）。
    *   **个人头像**：用户个人中心入口。

### 2.2 二级操作/筛选栏
*   **面包屑/标题**：显示“全部迭代”标题及图标。
*   **主操作按钮**：右侧蓝色按钮 **[＋ 新建]**。
*   **过滤器工具条**：
    *   **搜索框**：Placeholder显示“搜索 (⌘+G)”，用于按迭代名称搜索。
    *   **状态筛选**：下拉菜单，支持多选（如：未开始、进行中、已完成）。
    *   **负责人筛选**：下拉菜单，支持按用户筛选。
    *   **排序**：支持按创建时间、开始时间等字段排序。
    *   **统计信息**：显示当前列表下的总数量（如图中“58 个迭代”）。

### 2.3 数据列表区域
*   **展示形式**：以表格形式展示迭代详情。
*   **列表表头**：包含字段名称，部分字段支持点击排序。
*   **自定义列**：最右侧齿轮图标，允许用户配置展示的列字段。

## 3. 详细功能需求 (Functional Requirements)

### 3.1 列表展示字段

| 字段名称 | 数据类型 | 显示规则 | 交互/逻辑 |
| :--- | :--- | :--- | :--- |
| **名称** | 文本/链接 | 显示迭代名称（如“网络监控及告警”）。 | 点击名称跳转至该迭代的详情页/看板页。支持按名称排序。 |
| **状态** | 枚举 (Tag) | 不同状态显示不同背景色Tag：<br>1. **进行中** (黄色)<br>2. **已完成** (绿色)<br>3. **未开始** (通常为灰色，图中未显示) | 状态由系统自动流转或人工手动更改。 |
| **进度** | 进度条 + 百分比 | 1. **进度条**：绿色表示已完成比例，灰色表示剩余。<br>2. **数值**：显示百分比（如 30%）。 | 计算逻辑通常为：`已完成工作项 / 总工作项` 或 `已完成工时 / 总工时`。<br>*注：图中显示“已完成”状态的迭代进度不一定是100%，需允许此业务逻辑存在（即迭代结束但任务未全部关闭）。* |
| **类别** | 标签 (Tag) | 显示迭代类型，如“产品迭代”、“技术迭代”。无类别时显示“-”。 | 点击标签可快速筛选同类迭代（可选功能）。 |
| **负责人** | 用户对象 | 显示头像 + 姓名（如“方裕斌”）。 | Hover显示用户信息卡片；点击可筛选该负责人的所有迭代。 |
| **操作** | 按钮/菜单 | (图中未直接显示，通常悬停在行尾出现) | 编辑、删除、归档等快捷操作。 |

### 3.2 筛选与搜索逻辑

1.  **关键词搜索**：
    *   **范围**：仅检索“名称”字段。
    *   **触发**：输入字符后按回车，或实时过滤。
    *   **快捷键**：支持 `⌘+G` 聚焦搜索框。

2.  **状态筛选**：
    *   默认显示所有状态。
    *   支持多选（例如只看“进行中”的迭代）。

3.  **负责人筛选**：
    *   默认显示“全部”。
    *   支持搜索用户名称进行快速选择。

### 3.3 新建迭代功能

*   **触发**：点击右上角 **[＋ 新建]** 按钮。
*   **交互**：弹出侧边抽屉（Drawer）或模态框（Modal）。
*   **必填字段**：
    *   迭代名称
    *   所属项目（若跨项目）
    *   计划开始时间、计划结束时间
    *   负责人
*   **选填字段**：描述、目标、关联需求等。

### 3.4 排序逻辑
*   默认排序：通常按“状态”（进行中在前）或“最后更新时间”倒序排列。
*   支持点击表头“名称”进行升序/降序排列。
*   点击工具栏“排序”按钮可选择更多排序维度（如按进度高低）。

---

## 4. 业务逻辑与规则 (Business Rules)

1.  **状态与进度的关系**：
    *   **逻辑说明**：迭代的“状态”（Status）与任务的“完成进度”（Progress）解耦。
    *   **场景**：即便进度只有 94%（如“灰色发布”），管理员仍可以将迭代状态手动置为“已完成”。这通常意味着剩余的 6% 任务被移除、延期或作为遗留Bug处理。

2.  **权限控制**：
    *   **查看**：全员可见。
    *   **新建/编辑**：仅限项目经理 (PM)、Scrum Master 或管理员权限。
    *   **删除**：高权限操作，需二次确认。

3.  **空态处理**：
    *   若列表无数据，显示引导页：“暂无迭代，点击新建开始规划”。
    *   若字段（如类别）为空，显示短横线 “-”。

---

## 6. UI/UX 细节建议
*   **响应式**：在小屏幕下，优先展示“名称”、“状态”和“进度”，隐藏“类别”和“负责人”。
*   **视觉反馈**：进度条颜色应具有警示意义，若迭代时间过半但进度滞后，可考虑将进度条显示为橙色或红色（图中目前均为绿色，建议优化）。
*   **加载状态**：列表加载时应使用骨架屏（Skeleton Screen）过渡。
